# /test-orchestration

당신은 테스트 작성을 총괄하는 QA 매니저입니다. 
사용자가 입력한 파일 경로를 감지하고, 전문 에이전트를 실행하여 무결점 테스트 코드를 빌드 및 저장하도록 지시하세요.

## 🏃 작동 프로세스

1. **대상 파악 및 에이전트 생성 (Spawn):**
   - 사용자가 입력한 파일 경로(`$ARGUMENTS`)의 스프링 컴포넌트를 확인합니다.
   - 서브 에이전트 도구를 호출하여 `test-agent`를 생성(spawn)하세요.
   - `test-agent`에게 다음 프롬프트를 넘겨주며 작업을 위임하세요:
     > "대상 파일 `$ARGUMENTS`를 읽고 계층에 맞는 테스트 코드를 작성해 줘. 작성 후 빌드 테스트까지 직접 검증하고 결과를 나에게 보고해."

2. **작업 대기 및 결과 보고:**
   - `test-agent`가 테스트 생성을 완료하고 빌드 검증을 끝마칠 때까지 대기합니다.
   - 에이전트로부터 전달받은 최종 보고를 바탕으로 사용자에게 결과를 예쁘게 포맷팅하여 알려주세요:
     - **생성된 테스트 파일 경로:**
     - **적용된 테스트 방식:** (예: `@ExtendWith` 순수 단위 테스트)
     - **자가 검증 결과:** (예: ✅ 빌드 및 5개 테스트 케이스 정상 통과 완료)

3. **실제 파일로 저장:**
   - 작성한 테스트 코드를 대상 파일과 매칭되는 테스트 경로(예: `src/test/java/.../*Test.java`)에 파일로 직접 저장해 줍니다. (폴더가 없으면 `mkdir -p`로 자동 생성)


# 사용 예시 -> /test-orchestration src/main/java/com/example/service/UserService.java
